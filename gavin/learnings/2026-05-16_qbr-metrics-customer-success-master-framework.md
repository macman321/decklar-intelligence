## PART 1: THE QBR METRICS FRAMEWORK

### 1.1 Core Value Metrics Dashboard

Every QBR must include these 5 metric categories:

| Category | Primary Metrics | Data Source |
|----------|----------------|-------------|
| **Coverage** | % of shipments monitored vs. total shipped | Honeycomb API |
| **Visibility** | Real-time tracking %, alert response times | Platform analytics |
| **Protection** | Incidents detected, alerts fired, claims avoided | Customer claims data |
| **Compliance** | Cold chain integrity %, documentation accuracy | Sensor logs |
| **ROI** | Cost per shipment monitored, claims reduction $ | Customer financials |

### 1.2 The 4-Tier Value Hierarchy

**Tier 1 - Baseline (Months 1-3):**
- Shipment visibility established
- Alerting configured and validated
- Basic reporting functional
- Target: 95%+ tracking coverage

**Tier 2 - Optimization (Months 4-6):**
- Fine-tuned thresholds reducing false positives
- Proactive alert management
- Integration with customer workflows
- Target: <2% false positive rate

**Tier 3 - Intelligence (Months 7-9):**
- Predictive insights from pattern analysis
- Automated escalation workflows
- Custom dashboard adoption
- Target: 80%+ stakeholder adoption

**Tier 4 - Transformation (Months 10-12):**
- Supply chain optimization recommendations
- Cross-lane intelligence
- Expansion into new use cases
- Target: 3+ expansion opportunities identified

### 1.3 QBR Slide Structure (12 Slides Max)

1. **Executive Summary** (2-3 key wins, 1 flag, 1 ask)
2. **Deployment Health Score** (Green/Amber/Red with rationale)
3. **Usage Metrics Dashboard** (coverage, users, device types)
4. **Operational Performance** (alerts, incidents, response times)
5. **Value Realization** (quantified outcomes vs. baseline)
6. **Compliance & Quality** (cold chain integrity, data accuracy)
7. **Feature Adoption** (which capabilities are being used)
8. **Comparative Benchmarks** (vs. industry, vs. Decklar average)
9. **Success Criteria Review** (are we meeting stated goals?)
10. **Risk & Watch-Outs** (what could derail value)
11. **Expansion Opportunities** (new lanes, new use cases)
12. **Next 90 Days Action Plan** (specific commitments)

### 1.4 Critical Pre-QBR Checklist

**2 Weeks Before:**
- [ ] Pull all Honeycomb data for the quarter
- [ ] Validate metrics with customer service team
- [ ] Review previous QBR commitments
- [ ] Identify expansion opportunities

**1 Week Before:**

- [ ] Confirm attendees and their roles

**Day Of:**
- [ ] Arrive 15 min early, test screen share
- [ ] Have backup data in offline format
- [ ] Prepare answers to likely objections
- [ ] Bring device samples if relevant

## PART 2: CUSTOMER SUCCESS PLAYBOOKS

### 2.1 Health Scoring Methodology

**The Decklar Health Score (0-100):**

| Component | Weight | Calculation |
|-----------|--------|-------------|
| Deployment Maturity | 25% | Go-live age, configuration completeness |
| Usage Intensity | 25% | Active users, shipment volume, feature breadth |
| Support Burden | 20% | Tickets/month, escalations, response times |
| Value Realization | 20% | ROI metrics, expansion signals, referrals |
| Relationship Quality | 10% | NPS, executive engagement, communication cadence |

**Health Tier Actions:**

| Score | Tier | Action |
|-------|------|--------|
| 90-100 | 🟢 Champions | Accelerate expansion, ask for referrals, case study |
| 70-89 | 🟡 Healthy | Optimize usage, identify expansion, standard engagement |
| 50-69 | 🟠 At Risk | Intensive check-ins, address friction, executive escalation |
| <50 | 🔴 Critical | Emergency intervention, exec sponsor activation, save plan |

### 2.2 The 5-Stage Customer Journey

**Stage 1: Onboard (Weeks 1-4)**
- **Goal:** Fast time-to-value
- **Key Activities:** Portal training, first shipment monitoring, quick wins documentation
- **Success Metrics:** <30 days to first tracked shipment, 100% user onboarding
- **Risk Flags:** Delayed go-live, low user engagement, connectivity issues

**Stage 2: Activate (Months 2-3)**
- **Goal:** Daily usage habit formation
- **Key Activities:** Workflow integration, alerting fine-tuning, stakeholder expansion
- **Success Metrics:** 5+ shipments/week, <24hr alert response time
- **Risk Flags:** Sporadic usage, ignored alerts, user churn

**Stage 3: Optimize (Months 4-6)**
- **Goal:** Operational efficiency gains
- **Key Activities:** Threshold optimization, automation implementation, reporting standardization
- **Success Metrics:** 20%+ efficiency gain, proactive incident prevention
- **Risk Flags:** Stagnant usage patterns, unresolved pain points

**Stage 4: Expand (Months 7-9)**
- **Goal:** New use case adoption
- **Key Activities:** Cross-functional demos, new lane trials, advanced feature training
- **Success Metrics:** 2+ new use cases, 50%+ volume growth
- **Risk Flags:** Expansion stalled, competitive pressure, budget constraints

**Stage 5: Advocate (Months 10+)**
- **Goal:** Customer becomes reference
- **Key Activities:** Case study development, speaking opportunities, referral asks
- **Success Metrics:** Public endorsement, 1+ referral, renewal commitment
- **Risk Flags:** Leadership changes, strategic shifts, budget cuts

### 2.3 Intervention Playbooks

**Playbook: "The Go-Live Delayer"**
*Customer is 30+ days past planned go-live*

1. **Day 30:** Email + call to primary contact - "What do you need from us?"
2. **Day 37:** Escalate to project sponsor - blockers identification
3. **Day 45:** Executive-to-executive call - resource commitment
4. **Day 60:** "Success path" conversation - redefine scope if needed

**Playbook: "The Low User Adopter"**
*Platform is deployed but usage is minimal*

1. **Analyze:** Who has access? Who should have access? What's the workflow gap?
2. **Engage:** 1:1 training with non-users, identify workflow integration points
3. **Simplify:** Reduce friction (dashboard bookmarks, mobile setup, alert routing)
4. **Champion:** Find power user to evangelize internally
5. **Report:** Show executives what they're missing

**Playbook: "The At-Risk Renewal"**
*Customer showing disengagement signals 90 days before renewal*

1. **Assess:** Root cause analysis (price, value, competition, internal changes)
2. **Activate:** Executive business review - demonstrate quantified value
3. **Pivot:** Propose alternative (different tier, payment terms, expanded scope)
4. **Escalate:** Decklar executive involvement if needed
5. **Document:** Decision criteria and timeline

## PART 3: THE PHARMA-SPECIFIC FRAMEWORK

### 3.1 Pharma QBR Differentiators

**Required Metrics (vs. Standard QBR):**
- Temperature excursion frequency and severity (73-60-11 rule)
- GDP compliance documentation
- Regulatory audit readiness score
- Lane qualification status
- Excursion response time (target: <15 minutes)

**Required Slides (Additions):**
- Regulatory compliance dashboard
- Excursion drill-down (trends, root causes, resolutions)
- Lane risk heat map
- Audit readiness checklist status
- Quality agreement compliance

### 3.2 The 73-60-11 Rule Explained

**Framework for Temperature Deviation Analysis:**

- **73°F (23°C):** Standard room temperature threshold
- **60°F (16°C):** Cool/cold threshold - excursion risk zone
- **11:** Maximum acceptable excursion duration in minutes before escalation

**QBR Application:**
- Report % of shipments within 73-60 range
- Track excursions exceeding 11-minute threshold
- Trend analysis: Are excursions increasing/decreasing?
- Root cause categorization: Equipment, process, external factors

## PART 4: CONVERSATION FRAMEWORKS

### 4.1 The Value Conversation Structure

**Opening (5 min):**
"Since we went live [X months ago], you've monitored [Y shipments] across [Z lanes]. Here's what the data shows..."

**The Pivot (10 min):**
Present 3 specific value moments:
1. **Detection:** "In March, we caught a temperature excursion at 2:47 AM..."
2. **Prevention:** "Your proactive threshold adjustments have reduced..."
3. **Optimization:** "Lane analysis shows your Chicago route has 40% fewer..."

**The Expansion (10 min):**
"Based on what we've seen, there are 3 opportunities to extend this value..."
1. New lane expansion
2. Additional use case
3. Advanced feature adoption

**The Commitment (5 min):**
"For next quarter, let's commit to [specific metric improvement]. Here's what we'll each do..."

### 4.2 Handling Common Objections

| Objection | Response Framework |
|-----------|-------------------|
| "We don't have budget" | "Let's look at what you're already spending on [claims/inefficiencies] and where this saves money" |
| "We're considering competitors" | "Let's do a side-by-side value analysis. What specific capabilities are you evaluating?" |
| "Our team isn't using it" | "Let's understand why. Is it workflow friction, training, or value perception?" |
| "We need more time to evaluate" | "What's the specific concern? Let's create a 30-day validation plan with clear success criteria." |
| "It's too complex" | "Let's simplify. Which 3 features would deliver 80% of your value?" |
